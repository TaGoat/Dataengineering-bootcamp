Pipeline Architecture Diagram

Real-Time E-commerce Orders — Python → NiFi → Kafka → HDFS

1 Python Generator

Streaming GeneratorCreates CSV files every 1-3 seconds continuously
Messy DataMissing values, duplicates, corrupted rows, bad emails, mixed timestamps
streaming_data/orders_YYYYMMDD_HHMMSS_N.csv
CSV files

2 NiFi Producer Flow

ListFile → FetchFileDetects and reads new CSV files
UpdateAttributeAdds metadata (source, ingestion time, filename)
SplitTextChunks files at 64 KB max
ValidateRecordSchema and type validation
QueryRecordBusiness rules (required fields, positive values)
ConvertRecordCSV → JSON
PublishKafkaSends JSON to Kafka topic
JSON events

3 Apache Kafka

Topicecommerce_orders_json
Config3 partitions, replication factor 1, broker kafka:29092
Consumed JSON

4 NiFi Consumer → HDFS

ConsumeKafkaReads from topic, group: nifi-hdfs-consumer
UpdateAttributeBuilds date-partitioned HDFS path + UUID filename
PutHDFSWrites JSON to hdfs://itvlab:9000
/data/ecommerce_orders/year=YYYY/month=MM/day=DD/orders_uuid.json

Failure Handling

  • FetchFile → RetryFlowFile → retries exceeded → LogAttribute
  • SplitText / ValidateRecord / QueryRecord → LogAttribute
  • ConvertRecord / PublishKafka / PutHDFS → LogAttribute

Reliability

  • Back pressure queues prevent FlowFile buildup
  • Yield duration pauses processors after errors
  • Penalization delays failed FlowFiles before retry
  • Provenance tracking traces every FlowFile
